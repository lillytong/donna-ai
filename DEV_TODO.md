## Dev notes (operational gotchas — keep)

- **Dev backend must run with `uvicorn … --reload`**, else new routes/code aren't served by the running process (stale process → 404 on new endpoints). The CORS `expose_headers` change for export filenames also needs the app reloaded.
- **ALWAYS scope reload to `--reload-dir backend`** (`uvicorn backend.main:app --reload --reload-dir backend --port 8000`). A bare `--reload` watches the WHOLE repo, including `.claude/worktrees/agent-*/` where isolated-worktree agents edit files — so every agent save churns/restarts the live server and intermittently breaks the running app (a `--reload-exclude '.claude/*'` works too). Worktree isolation isolates git state, NOT the filesystem watch. Bit us 2026-06-25 (Donna recommendations appeared "down" mid-F03c-build).
- **Generation surfaces' `max_tokens` ≥ 4096** (`donna_recommendation`/`clause_draft`/`chat_advise`/`brainstorm_chat`). At 1024 a complex issue's recommendation JSON truncates mid-object → parse fails → operator sees the "couldn't produce a grounded recommendation" fallback. max_tokens is a cap not a target, so short outputs are unaffected. Fixed `9ad060a`.

- **`tsc --noEmit` does NOT catch CSS-module path errors — the frontend gate needs more.** Next ships a global `*.module.css` type shim, so `import styles from "./whatever.module.css"` type-checks clean *even if the file doesn't exist or the path is wrong* → tsc passes, the webpack build fails at runtime with "Module not found." Bit us 2026-06-25 (F03c review page imported `./review.module.css` but the file was one dir up → build error the operator hit live). **Gate fix:** after any frontend build that adds/moves a CSS module (or any non-TS asset import), additionally verify EITHER (a) every `*.module.css` import resolves to a real file (`grep` the imports, confirm each path exists relative to its importer), OR (b) a real compile — the running `next dev` recompile of the route, or `next build` in an ISOLATED worktree (never `next build` against the operator's live `.next`). tsc alone is necessary, not sufficient, for asset imports.
- **Build subagents can corrupt ASCII quotes → Unicode smart quotes (breaks tsc/compile); recover deterministically, don't redo.** 2026-06-26: the frontend subagent's edits turned `"..."`/`'...'` into `“...”`/`’` throughout `page.tsx` (TS1127 "Invalid character") — and the watchdog killed it mid-edit, leaving partial work. When an agent stalls (watchdog, no-progress), ALWAYS `git status`/`diff` + compile-check the touched files before trusting or continuing — assume partial/broken state.

  **The problem, precisely.** Given baseline tree `B={b_i}` (the `last_shared_with_counterparty` `SnapshotNode`s — stable ids, parent_id, order_index, heading, body; `snapshot.py`) and a freshly parsed incoming tree `R={r_j}` (NO ids), produce an **injective partial map `M: R → B ∪ {NEW}`**; baseline nodes left unmatched = DELETED. `M` is exactly what fills `counterparty_revision_changes.node_id` (NULL = NEW, + `proposed_parent_id`/`proposed_order_index`) and is what carries DD-28/DD-54 **classification inheritance** (a matched `r` inherits `b`'s role/has_placeholder; only NEW nodes get classified).   **The oracle (two layers — mirrors round-trip `tests/system/test_export_roundtrip.py` + `evals/clause_search/`):**
  - **Layer A — mechanical invariants, no ground truth, deterministic (the free round-trip-style gate):** (1) **injectivity** — no `b` claimed twice; (2) **partition completeness** — `|R| = matched + NEW`, `|B| = matched + DELETED`, counts reconcile; (3) **reconstruction round-trip (inverse-pair oracle)** — apply `M`'s implied diff to `B` (accept all NEW/DELETE + matched-pair text swaps) → reconstructed tree must equal the parsed incoming tree byte-for-byte; dropped/duplicated content diverges here; (4) **self-match no-op** — matching a document against ITSELF yields zero NEW/DELETED/edited (catches threshold + normalization bugs; the synthetic always-runs fixture).
